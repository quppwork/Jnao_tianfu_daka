## 一、核心流程

```
每日开始 → 生成今日训练报告（简洁指令）
         → 学员逐项完成训练（顺序执行，不可跳）
         → 每完成一项立即打卡（不能攒到最后一起打）
         → 打卡数据记录 → 作为下次训练的承接基础
```

---

## 二、关键机制

### 2.1 顺序执行
- 的一项一项做，做完一项→打卡→下一项
- 不能一次性打完所有卡（细节会乱）

### 2.2 学习连续性
- 今天的方案基于昨天的训练内容推演
- 学习不是每次无关，必须有承接关系

### 2.3 每日训练报告
- 训练开始前生成
- 内容极简（越少越好）
- 相当于"今日指令"

### 2.4 智能时长推荐
```
今天计划时间 ≠ 昨天计划时间
  → AI 询问："多出了 X 小时，想重点练什么？"
      ├─ 学员有需求 → 按指定安排
      └─ 学员没需求 → AI 推荐薄弱项目
```

---

## 三、清北班 vs 非清北班

| 区别 | 清北班 | 非清北班 |
|------|--------|---------|
| 打卡后 | 导师需审核通过 | 直接通过 |
| 提交人 | 学员本人 | 可能是家长或学生 |

---

## 四、每项训练的内容

- 听什么音频
- 看什么视频
- 指导建议和注意事项
- 训练时长和内容量
- 打卡要求

---

## 五、与现有系统的差距

| 现有实现 | 需要改/加 |
|---------|----------|
| 训练 A→B 两步结构 | 改为 N 项顺序链 |
| 多选能力打卡 | 改为逐项顺序打卡 |
| 无每日报告 | 加 AI 生成每日报告 |
| 无时长对比 | 加智能时长推荐 |
| 无导师审核 | 清北班加审核步骤 |
| 无学习连续性 | 加历史承接逻辑 |

## 六、待确认问题

1. **每日训练报告谁生成？** 老板手动配课表，还是 AI 自动生成？
2. **训练项的顺序和内容由谁定义？** 老板预先配置，还是 AI 动态安排？
3. **现有 12 个能力类型是否保留？** 还是替换为老板定义的具体训练项？
4. **导师审核在 App 内完成还是外部？**
5. **每日计划训练时长从哪里来？** 学员自己填、老板预设、还是系统根据等级自动定？
