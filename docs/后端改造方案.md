## 五、数据迁移

老 `training_progress` 结构：

```json
{
  "main_line": "A",
  "skills": { "超脑阅读": { "stage": 1, "part": 1 } },
  "main_line_sessions": 5,
  "pending_main_line_to": null,
  ...
}
```

迁移到新结构：

```python
def migrate_old_state(old: dict) -> dict:
    """老 state → 新 state 迁移（一次性）"""
    return {
        "overall_tier": 1,                        # 全部从 Tier 1 开始
        "skills": {
            "超脑阅读": {"tier": 1, "oss_stage": 0, "oss_part": 0, "consecutive_pass": 0},
            "影像追忆": {"tier": 1, "oss_stage": 1, "oss_part": 1, "consecutive_pass": 0},
            "扫描速记": {"tier": 1, "oss_stage": 1, "oss_part": 1, "consecutive_pass": 0},
            "极速运算": {"tier": 1, "oss_stage": 2, "oss_part": 1, "consecutive_pass": 0},
            "极速学习": {"tier": 1, "oss_stage": 2, "oss_part": 1, "consecutive_pass": 0},
        },
        "training_days": old.get("training_days", 0),
        "training_day_anchor": old.get("training_day_anchor"),
        "last_settled_plan_date": old.get("last_settled_plan_date"),
    }
```

## 六、扩展预留

当前版本仅实现 Tier 1，但架构预留了 Tier 2+ 的扩展点：

| 扩展点 | 实现方式 |
|--------|---------|
| 新增 Tier N 的时长公式 | 在 `duration_formula` 中加 `tier: N` 条目 |
| 新增 Tier N 的达标标准 | 在 `tier_thresholds` 中加对应 Tier 的阈值 |
| 新增技能（如文科奥秘） | 在 `skills.required` 加技能名 + tier_thresholds 加阈值 |
| 技能在某个 Tier 退役 | 公式中不包含该技能即可（代码自动跳过） |
| 不同天赋不同公式 | `duration_formula` 按 `talent_code` 分表 |
