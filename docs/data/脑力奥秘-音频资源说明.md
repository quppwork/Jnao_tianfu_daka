## 1. 和后台截图的关系

你提供的后台分类：

```

├── 思 (32)

├── 德 (31)

```

**本地数据库 dump（2026-06-17）中**，与「五天赋」直接对应、且为 **MP3 音频** 的数据在：

| 表 | 说明 |
|----|------|
| `ys_c_classify` | 大类 id=19「我知道你是谁」 |
| `ys_c_course` | 五门系列课，字段 **`talent`** 存天赋标签 |

若线上后台显示 **162 条**，可能原因：

1. 线上 RDS 比本地 dump 更新，新增了内容  
2. 后台「脑力奥秘」还统计了其他类型（视频、其他分类下的同名标签）  
3. 分类树与 `ys_c_course` 不是同一张配置表  

**MVP 建议**：先用本文件 41 条 MP3 跑通闭环；老板确认 162 条来源后再做第二次同步。

## 3. 播放 URL 字段

| 表.字段 | 内容 |
|---------|------|
| `ys_c_av.aliyun_id` | **主播放地址**（虽是字段名 aliyun_id，实际多为完整 HTTPS URL） |
| `ys_c_av.url` | 多为 NULL，以 `aliyun_id` 为准 |
| `ys_c_av.sort` | 课时顺序（推送按此排序） |
| `ys_c_av.is_try` | 是否试听 |
| `ys_c_av.status` | 1=启用 |

URL 域名示例：

- `https://video.jnao.com/sv/{id}/{id}.mp3` — 五天赋系列音频（本 MVP 使用）

## 4. SQL：自己查库

```sql
-- 五天赋 MP3 全量
SELECT
  c.talent,
  CASE c.talent
    WHEN 1 THEN '学' WHEN 2 THEN '思' WHEN 3 THEN '行'
    WHEN 4 THEN '德' WHEN 5 THEN '赢'
  END AS tag,
  c.title AS course_title,
  a.sort,
  a.title AS lesson_title,
  a.aliyun_id AS play_url
FROM ys_c_av a
JOIN ys_c_course c ON c.id = a.course_id
WHERE c.classify_id = 19
  AND c.talent IN (1, 2, 3, 4, 5)
  AND a.status = 1
  AND a.aliyun_id LIKE '%.mp3'
ORDER BY c.talent, a.sort, a.id;
```
