## 一、整体框架（已理解）

```
三层体系：

  ┌─────────────────────────────────────────────────┐
  │  第一层：阶位（Tier 1-9）                          │
  │  决定：练哪些技能、达标标准是什么                      │
  ├─────────────────────────────────────────────────┤
  │  第二层：学段（1-3年级 / 4-6年级 / 初中 / 高中）        │
  │  决定：同一阶位内不同年级的具体阈值                     │
  ├─────────────────────────────────────────────────┤
  │  第三层：每日训练安排（时长公式）                       │
  │  决定：今天练什么技能、每个技能练几轮                    │
  └─────────────────────────────────────────────────┘

三条推进线（互不等待）：
  ┌──────────────────┐  ┌──────────────────────┐  ┌────────────────────┐
  │  线1：每日训练安排  │  │  线2：技能 Tier 独立晋级│  │  线3：整体 Tier      │
  │  今天练 a+b+c     │  │  影像追忆 Tier 1→2    │  │  所有活跃技能中      │
  │  由时长公式决定    │  │  需连续3次达标          │  │  取最低 Tier         │
  │  暂统一用Tier1公式 │  │  各走各的互不等待       │  │  = 最低原则          │
  └──────────────────┘  └──────────────────────┘  └────────────────────┘
         ↓                       ↓                        ↓
    今天生成什么训练项         该技能达标标准升级         决定排课用哪个公式（暂同）
```

### 关键设计原则（已确认）

1. **技能独立晋级**：每个技能的 Tier 单独判定，连续 3 次达标 → Tier+1，各走各的
2. **整体 Tier = 最低原则**：取所有活跃技能中最低的 Tier。例如超脑=5、扫描=1 → 整体 Tier=1
3. **训练先行、考核滞后**：每天练的比晋级考的宽，扫描速记 Tier 1 就开始练，Tier 2 才纳入考核
4. **OSS Stage 不绑定 Tier**：OSS 的 stage 编号仅用于"今天取哪个音频"，按 OSS 实际存量推进，有则取、无则跳。可用 OSS 最高 stage/part 耗尽时，该技能的 OSS 推进终止
5. **音频不变只改标准**：超脑阅读单音频，Tier 晋级 → 达标标准变，音频不变
6. **目前仅 Tier 1 公式**：Tier 2+ 每日训练公式老板未提供，暂全用 Tier 1 公式

## 四、每日训练分配公式（Tier 1，已理解）

### 4.1 时长 → 训练项映射

```
时长        小学                          初高中
────────────────────────────────────────────────────────
20min      A                             A 
40min      A + B                         A + B
1-2h       A + B + C                     A + B + C（不建议）
2-3h       A + 2B + C + 高效作业          A + 2B + C（不建议）+ 高效作业
           （≥3阶：高效作业→E）              （≥3阶：高效作业→E）
           建议2套以上试卷                   建议2套以上试卷
3-4h       A + 2B + C + 高效作业          A + 2B + C（不建议）+ 高效作业
           （≥3阶：高效作业→2E）             （≥3阶：高效作业→2E）
           建议3套以上试卷                   建议3套以上试卷
4-5h       A + 3B + C + 高效作业          A + 3B + C（不建议）+ 高效作业
           （≥3阶：高效作业→3E）             （≥3阶：高效作业→3E）
           建议4套以上试卷                   建议4套以上试卷
```

**符号说明**：

- A = 超脑阅读 新人标准任务（1轮）
- B = 影像追忆 新人标准任务（1轮）
- C = 扫描速记 新人标准任务（1轮）
- D = 极速运算（三阶才出现）
- E = 极速学习（≥3阶后替换高效作业）
- 高效作业 = 选修音频
- "试卷" = 用户自己准备的线下试卷，系统仅提醒

**关键规则**：

- "不建议" = 展示但标注文案，用户可自行选择
- "≥3阶替换" = 高效作业的 slot 变为极速学习

## 五、Stage/Part 推进规则（已理解）

### 5.1 单个技能的推进

```
技能 X 当前 (stage=N, part=M)

每天打卡该技能 → 产生打卡卡片
  → 判断是否达标（按阶位+学段的量化标准）
    → 达标 → consecutive_pass += 1
    → 达标 + consecutive_pass ≥ 3
      → 推进到下一个 (stage, part)
      → OSS 池中查找：先从同 stage 找 part+1；找不到则 stage+1, part=1
      → consecutive_pass = 0（重置）
    → 不达标 → consecutive_pass = 0（严格连续，中断重置）
```

### 5.2 技能起始 Stage 由天赋决定

```
极速学习 OSS first_stage:
  学者(1): stage 2
  思者(2): stage 2
  行者(3): stage 3   ← 更高起点
  德者(4): stage 3   ← 更高起点
  赢者(5): stage 2

极速运算 OSS first_stage:
  五者统一: stage 2

影像追忆 OSS first_stage:
  五者统一: stage 1

扫描速记 OSS first_stage:
  五者统一: stage 1

超脑阅读:
  五者统一: 单文件，无 stage/part
```

**核心原则**：OSS 实际存储的 stage/part 就是教学序列，天赋决定了"从哪个 stage 开始"。

## 七、晋级机制（✅ 已确认）

### 7.1 技能 Tier 独立晋级（核心）

每个技能**独立判定** Tier 晋级，互不等待：

```
技能 X 当前 Tier=N，打卡卡片达标：
  → 达标 → consecutive_pass += 1
  → 达标 + consecutive_pass ≥ 3 → 该技能 Tier 升到 N+1，consecutive_pass 重置为 0
  → 不达标 → consecutive_pass 重置为 0（严格连续，中断重置）
```

**各技能 Tier 不同步**：超脑阅读可能已经 Tier 3，扫描速记还在 Tier 1。

### 7.2 整体 Tier = 最低原则（✅ 已确认）

```
整体 Tier = min(所有活跃技能的 Tier)

示例：
  超脑 Tier=5, 影像 Tier=2, 扫描 Tier=1 → 整体 Tier = 1
  超脑 Tier=5, 影像 Tier=5, 扫描 Tier=1 → 整体 Tier = 1
  超脑 Tier=2, 影像 Tier=2, 扫描 Tier=2 → 整体 Tier = 2
  超脑 Tier=3, 影像 Tier=2, 扫描 Tier=1 → 整体 Tier = 1
```

采用"最低原则"而非"50%原则"的原因：只要有一个技能落后，就应该迁就它继续训练。整体 Tier 目前仅用于决定排课公式（暂统一用 Tier 1），后续可扩展为不同 Tier 使用不同公式。

### 7.3 OSS Stage/Part 推进（✅ 已确认）

**OSS Stage 和 Tier 是两套独立体系**：

```
│  推进                                    │  推进
│  连续3次达标 → part+1 或 stage+1           │  连续3次达标 → Tier+1
│  按 OSS 实际存量走，有则取、无则跳            │  达标标准随之升级
│  耗尽 → 该技能不再推送音频（"不练了"）         │  Tier 仍可继续升级（只改标准不改音频）
└──────────────────────────────────────┘

两者不绑定。例如超脑阅读：OSS 单文件无 stage → OSS 推进无效，只改 Tier 标准。
```

**OSS 推进规则（代码逻辑）**：

```python
def bump_oss_stage(skill, talent_code, current_stage, current_part):
    pool = get_skill_lessons(skill, talent_code)  # 该天赋该技能的 OSS 音频列表
    # 找同 stage 下一个 part
    next_lesson = pool.find(stage=current_stage, part=current_part + 1)
    if next_lesson:
        return (current_stage, current_part + 1)
    # 找下一个 stage 的第一个 part
    next_lesson = pool.find(stage=current_stage + 1, part=1)
    if next_lesson:
        return (current_stage + 1, 1)
    # 耗尽
    return None  # 该技能 OSS 池已空
```

**天赋差异**（以极速学习为例）：


| 天赋       | first_stage | OSS 路径                             |
| -------- | ----------- | ---------------------------------- |
| 学者/思者/赢者 | 2           | 2-1→3-1→3-2→4-1→4-2→5-1→5-2→5-3→耗尽 |
| 行者/德者    | 3           | 3-1→3-2→4-1→4-2→5-1→5-2→5-3→耗尽     |


行者/德者跳过 stage 2 是因为 OSS 中没有他们对应的音频文件，不是人为跳过。

### 13.2 核心流程伪代码

```
POST /api/training/schedule { planned_minutes: 120 }

1. 获取用户 state（含所有技能 tier + oss_stage/part）
2. 计算 overall_tier = min(all_skill_tiers)
3. 用 overall_tier 对应的时长公式生成训练项列表：
   Tier1公式: [A, B, C, 高效作业]（按分钟数展开份数）
4. 遍历公式槽位，为每个技能取当前 OSS (stage,part) 对应的音频
   - 有音频 → 生成 TrainingItem
   - 无音频 → 占位或跳过
5. 选修技能单独标记（高效作业/多元感知/精力恢复）
6. 返回训练方案

POST /api/training/checkin { item_id, cards }

1. 记录打卡数据
2. 解析 cards → 提取技能名 + 打卡值（字数/用时/准确度）
3. 按技能 Tier × 学段查找达标阈值
4. 判断达标：
   - 达标 → consecutive_pass += 1
   - 不达标 → consecutive_pass = 0
5. 判断 Tier 晋级：
   - consecutive_pass ≥ 3 → 技能 Tier += 1, consecutive_pass = 0
6. 判断 OSS 推进：
   - 达标 → 查找下一个可用 (stage, part) → 找到则推进
   - OSS 耗尽 → 标记该技能"OSS已完成"
7. 重新计算 overall_tier = min(all_skill_tiers)
8. 返回：各项技能达标状态、Tier变化、OSS推进状态
```
