## 一、系统定位

打卡系统是 JNAO 天赋成长平台的四大核心模块之一，承载用户"每日训练"功能。用户在完成天赋测评后，通过每日打卡训练持续提升能力，训练数据与成长里程碑模块联动。

### 四大模块关系

```
┌──────────┬──────────┬──────────┬──────────┐
│ 天赋测试  │ 今日训练  │ 学科答疑  │ 成长里程碑 │
│  (测评)   │  (打卡)   │  (问答)   │  (追踪)   │
└──────────┴──────────┴──────────┴──────────┘
      ↑         ↑          ↑          ↑
      └─────────┴──────────┴──────────┘
                共用后端 API
                共用用户体系
                共用 MySQL 数据库
```

## 三、状态流转

### 3.1 训练 A → 训练 B 的解锁逻辑

```
初始状态                    点击"训练 A 打卡"后           刷新页面后
────────                   ────────────────           ──────────

训练 A:  正常显示           训练 A:  已完成              训练 A:  正常显示
训练 B:  虚化（opacity:0.35） 训练 B:  完全显示            训练 B:  虚化（恢复初始）
         pointer-events:none          pointer-events:auto
         文字灰色                       文字正常色
         显示"⚠ A 未完成"              显示"✅ A 已完成，
                                         开始训练 B"
```

### 3.2 关键规则

| 规则 | 说明 |
|---|---|
| **顺序解锁** | 训练 A 未完成 → 训练 B 不可操作（虚化+禁用点击） |
| **打卡触发** | 点击"训练 A 打卡"按钮 → B 解锁 + 总结生成 |
| **不持久化** | 刷新页面后恢复初始状态（本地预览阶段） |
| **通用命名** | 训练内容统一标记为"视频训练/音频训练"，不绑定具体内容 |

## 四、数据设计（待开发）

### 4.1 数据库表结构（草案）

```sql
-- 训练方案表
CREATE TABLE training_plan (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    user_id     INT NOT NULL,
    train_date  DATE NOT NULL,
    plan_label  VARCHAR(100),         -- 如"方案：训练内容+验收标准"
    created_at  DATETIME DEFAULT NOW()
);

-- 训练步骤表
CREATE TABLE training_step (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    plan_id     INT NOT NULL,
    step_order  INT NOT NULL,         -- 1, 2, ...
    step_type   VARCHAR(20) NOT NULL, -- 'video' | 'audio'
    step_label  VARCHAR(100),         -- 如"视频训练"
    content_label VARCHAR(100),      -- 如"训练用视频"
    duration    INT,                  -- 预计时长（分钟）
    is_completed BOOLEAN DEFAULT FALSE,
    completed_at DATETIME,
    FOREIGN KEY (plan_id) REFERENCES training_plan(id)
);

-- 打卡记录表
CREATE TABLE checkin_record (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    user_id     INT NOT NULL,
    plan_id     INT NOT NULL,
    checkin_date DATE NOT NULL,
    summary     TEXT,                 -- AI 生成的训练总结
    created_at  DATETIME DEFAULT NOW(),
    FOREIGN KEY (plan_id) REFERENCES training_plan(id)
);
```

### 4.2 后端 API 设计（草案）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/training/today` | 获取今日训练方案（含 A/B 状态） |
| POST | `/api/training/step/{id}/complete` | 标记某步骤完成 |
| POST | `/api/training/checkin` | 提交打卡，生成总结 |
| GET | `/api/training/history` | 获取历史打卡记录 |

## 六、与其它模块的联动

| 联动方向 | 说明 |
|---|---|
| **天赋测试 → 打卡** | 测评结果决定推荐的训练难度/类型 |
| **打卡 → 成长里程碑** | 连续打卡天数触发里程碑解锁（如 7 天/30 天/100 天） |
| **打卡 → 学科答疑** | 训练中遇到的问题可直接跳转答疑模块 |
| **打卡 ↔ 用户体系** | 打卡记录关联用户 ID，用于历史追踪 |
