> 最后更新：2026-06-29

## 一、功能定位

AI 一对一学科辅导。用户通过语音或文字提问，AI 以语音+文字回复，伴随虚拟老师形象。

核心体验：**像跟一位老师面对面聊天一样自然。**

## 三、"不会"引导流程（核心设计）

用户完全不知道该问什么时，系统用三级引导把用户带上路：

### 3.1 引导流程图

```
用户："不会"
  ↓
一级引导 — 选学科
  AI："没关系！先告诉我是什么学科？"
  [📐数学] [📖语文] [🌍英语] [🔬科学] [❓其他]
  ↓ 用户点击 [📐数学]
  ↓
二级引导 — 选题型
  AI："好的，数学哪里不会？"
  [🧮计算题] [📊应用题] [📐几何] [📏测量] [❓说不清楚]
  ↓ 用户点击 [❓说不清楚]
  ↓
三级引导 — 拍照/描述
  AI："没事，你拍个照片给我看看题目，
       或者试着描述一下题目里有什么字？"
  [📷 拍照发题] [🎤 描述题目]
  ↓
进入正常问答
```

### 3.2 设计原则

- **每一级都给可点击按钮**，用户可以零输入走到最后
- **"说不清楚"是一个合法选项**，不强迫用户打字
- **拍照是最后的兜底**，家长可以帮拍题目

## 四、异常场景处理

| 场景 | 用户行为 | 系统处理 |
|------|---------|---------|
| 沉默 | 点了录音，不说话 | "没有听到声音，可以再说一次吗？" 3秒后提示 |
| 太短 | 说"嗯""啊" | "好像没听清楚，能不能多说一点？" |
| 太吵 | 背景有电视/人声 | "周围有点吵，换个安静的地方试试？" |
| 说脏话 | 骂人 | "这样说不太好哦。有什么题目我帮你解决？" |
| 情绪低落 | "学习好烦" | 安抚 + 引导："是不是卡住了？说来听听，我帮你看看" |
| 非学科问题 | "我想玩游戏" | "做完一道题就能休息了～现在先试试这道？" |
| 超纲 | 问了 AI 不懂的 | "这道题超出我的能力了，我帮你记下来发给老师？" |
| 网络断 | 录音上传一半断了 | "网络好像不太好，重新说一次？" |
| 连续三次不懂 | AI 解释了三次用户还是说"不会" | "要不要换一题练练手？或者休息一下再回来？" |
| 安全问题 | "我想跳楼" | **高危警报**。回复预设安全话术 + 建议联系家长/老师。不提供 AI 自由回答 |

## 六、技术方案

### 6.1 语音输入链路

```
用户说话 → 火山引擎 ASR → 文字 → 火山引擎/DeepSeek → AI 回复
                                                          ↓
                                             文字显示 + 火山引擎 TTS 语音播报
```

### 6.2 组件选型

| 功能 | 方案 | 状态 |
|------|------|------|
| AI 对话 | 火山引擎豆包 / DeepSeek（已有） | 后端代理，前端不直接调 |
| 语音识别 ASR | 火山引擎 ASR | 需申请 API |
| 语音合成 TTS | 火山引擎 TTS（可克隆张宇老师声音） | 需采集声音样本 |
| 虚拟形象 | 2D 图片 + 呼吸脉冲动画 | 第一阶段方案 |
| 文字输入 | 保留，用户可选择 | 必须保留 |

### 6.3 API 设计（草案）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/qa/chat` | 文字对话（同步） |
| POST | `/api/qa/chat/stream` | 文字对话（SSE 流式） |
| POST | `/api/qa/voice` | 语音输入：上传音频 → 返回 AI 文字回复 + TTS 音频 |
| GET | `/api/qa/history` | 获取对话历史列表 |
| POST | `/api/qa/session` | 新建对话 |
| GET | `/api/qa/session/{id}` | 获取对话详情（消息列表） |
| DELETE | `/api/qa/session/{id}` | 删除对话 |

### 6.4 数据库表（草案）

```sql
CREATE TABLE qa_session (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    user_id     INT NOT NULL,
    title       VARCHAR(200),
    subject     VARCHAR(20),       -- 'math'|'chinese'|'english'|'science'|'other'
    created_at  DATETIME DEFAULT NOW(),
    updated_at  DATETIME DEFAULT NOW()
);

CREATE TABLE qa_message (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    session_id  INT NOT NULL,
    role        VARCHAR(10) NOT NULL,  -- 'user' | 'assistant'
    content     TEXT NOT NULL,
    voice_url   VARCHAR(500),         -- 语音消息的音频URL
    created_at  DATETIME DEFAULT NOW(),
    FOREIGN KEY (session_id) REFERENCES qa_session(id)
);
```
