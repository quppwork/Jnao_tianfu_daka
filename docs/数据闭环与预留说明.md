# 数据闭环与预留能力说明

> 最后更新：2026-06-29  
> 与产品拍板一致：已实现项写清代码位置；预留项仅保留 schema/文档；未立项能力从主文档移除。

---

## 一、已实现闭环

### 1. 打卡效果/备注 → 次日 AI 方案（Q1 ✅）

用户在前端打卡填写的 `result`（训练效果）、`note`（备注）会进入昨日训练摘要，供豆包生成今日报告与排课参考。

| 环节 | 说明 |
|------|------|
| 写入 | `POST /api/training/checkin` → `training_record.result/note` 及 `files_json[].result/note` |
| 读取 | `training_service.get_yesterday_training_context()` |
| 消费 | `training_plan_generator.generate_daily_report_text()`、`training_route_context.build_route_context()` |

### 2. 答疑易错点/教练提示 → 后续对话（Q2 ✅）

| 环节 | 说明 |
|------|------|
| 写入 | `qa_service` 每次回复将 `coach_hint`、`mistake_pattern` 写入 `qa_message.meta_json` |
| 读取 | `qa_coach.fetch_recent_coach_context_for_prompt()` |
| 消费 | `build_qa_system_prompt(..., coach_context=...)` 注入系统提示 |

## 二、预留（有字段/能力，暂不开发）

| 能力 | 位置 | 说明 |
|------|------|------|
| 打卡复核 | `training_record.review_status` 等 | 默认 `approved`，无审核 API；B 端上线时再补状态机 |
| 首页引导个性化 | `guide_service` / `guide_session` | 当前静态人设；后续可读 `profile_json`、训练进度 |
| 成长达成日精确回溯 | `growth_service._build_earned_at_map` | 当前部分徽章用查看日；后续按打卡记录回溯 |
| 答疑语音消息 | `qa_message.voice_url` | 当前仅文本+图片；语音答疑立项后再用 |
| 清北班标记 | `child_user.is_qingbei` | 仅 profile 返回，无专属训练逻辑 |

---

## 三、当前不做（已从主规格移除）

以下能力**不在当前版本范围**，文档与排期中不再作为待办：

- 训练记录运营审核后台（在未做 Q3 实现前不单独规划）
- `media_exhausted` 后台定时主动结算（仍由进页 `GET /today` 被动更新）
- 主线进阶用户侧预告/推送（`pending_main_line_to` 仅次日静默生效）
- 测评 `attributeList` 全量维度雷达图 API（前端报告页已用 JNAO 报告）
- `TalentAssessmentArchive` 查询/恢复接口
- `content_item.source_id/course_id` 对账自动化
- `jnao_uid` 与 JNAO 平台反查校验
- `time_spent` 历史趋势分析
- 视频/音频偏好自动排课
