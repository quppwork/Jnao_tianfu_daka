## 一、核心业务模型

将训练系统拆解为 4 个核心概念，形成「评估→规划→执行→反馈」闭环：

```mermaid
graph LR
    A["🎯 评估<br/>Assessment"] --> B["📋 规划<br/>Planning"]
    B --> C["🏃 执行<br/>Execution"]
    C --> D["📊 反馈<br/>Feedback"]
    D --> A

    style A fill:#e1f5fe,stroke:#0288d1
    style B fill:#e8f5e9,stroke:#388e3c
    style C fill:#fff3e0,stroke:#f57c00
    style D fill:#fce4ec,stroke:#d81b60
```

| 概念 | 英文 | 职责 | 现有实现 |
|------|------|------|----------|
| **训练阶段** | Phase | 定义阶段 A~E，每个阶段有准入标准和毕业标准 | `training_curriculum.yaml` + `training_advance_rules.yaml` |
| **训练项目** | Task/Module | 阶段内的具体课程，有类型（超脑阅读/影像追忆/扫描速记…）、前置依赖、单轮时长 | `ContentItem` 表 + `content_meta.py` |
| **训练记录** | Record | 孩子昨日完成情况（完成度、用时、质量分） | `TrainingRecord` 表 + `TrainingItem` 表 |
| **全局约束** | Constraint | 今日可用时长、每日最大训练量、解锁时间（凌晨4点） | `TrainingWindow` + `training_day.py` |

## 二、闭环执行流程

```mermaid
flowchart TD
    START(["🌙 每日凌晨 4:00 触发"]) --> S1

    subgraph S1["1️⃣ 状态读取"]
        A["读取 child_training_state<br/>当前阶段 / 当前项目 / 第几轮"]
    end

    S1 --> S2

    subgraph S2["2️⃣ 昨日复盘（反馈）"]
        B{"昨日是否完成？"}
        B -->|"❌ 未完成"| C["今日课程 = 昨日未完成项<br/>（重训练 / continue）"]
        B -->|"✅ 已完成"| D{"是否达到毕业标准？<br/>（正确率 > 80% / 轮次达标）"}
        D -->|"✅ 达标"| E["项目推进：<br/>下一项目 / 下一阶段"]
        D -->|"❌ 未达标"| F["辅助训练：<br/>调取前置基础项目补强"]
    end

    S2 --> S3

    subgraph S3["3️⃣ 资源规划（背包算法）"]
        G["拿到待做项目列表"]
        G --> H["根据用户设定时长（30/60/90/120 min）"]
        H --> I["贪心填充 / 0-1 背包<br/>选出最优训练单元组合"]
    end

    S3 --> S4

    subgraph S4["4️⃣ 依赖校验"]
        J["检查：先看后练 ✓"]
        J --> K["检查：前置项目已完成 ✓"]
        K --> L["检查：A→B 顺序 ✓"]
    end

    S4 --> END(["📱 生成今日方案 → 返回前端"])

    style START fill:#e8eaf6,stroke:#3f51b5
    style END fill:#e8f5e9,stroke:#4caf50
```

### 关键规则

| 规则 | 说明 | 现有实现 |
|------|------|----------|
| **昨日未完 → 重训** | 昨日未打卡，今日延续同一课程 | `training_service.py: _continue_incomplete` |
| **达标 → 推进** | 项目完成且质量达标，进入下一个项目 | `training_advance_rules.yaml` + `training_mastery.py` |
| **未达标 → 补强** | 调取前置阶段基础项目做辅助训练 | `training_optional_service.py` |
| **A→B 顺序** | 必须先完成 A 方案所有项才能打卡 B | `training_block_builder.py` |
| **凌晨4点解锁** | 新的一天从凌晨4点算起 | `training_day.py` |

## 四、状态机设计

```mermaid
stateDiagram-v2
    [*] --> Freshman: 新用户注册

    state Freshman {
        [*] --> NoAssessment: 等待天赋测评
        NoAssessment --> Assessed: 完成天赋测试
    }

    Assessed --> Phase_A: 进入阶段A（基础期）

    state Phase_A {
        [*] --> A_Today: 生成A方案
        A_Today --> A_Checkin: 逐项打卡
        A_Checkin --> A_Complete: A方案完成
        A_Complete --> B_Today: 解锁B方案
        B_Today --> B_Checkin: 逐项打卡
        B_Checkin --> DayComplete: 当日完成
    }

    DayComplete --> NextDay: 次日4:00解锁
    NextDay --> Phase_A: 继续/推进

    Phase_A --> Phase_B: 毕业达标
    Phase_B --> Phase_C: 毕业达标
    Phase_C --> Phase_D: 毕业达标
    Phase_D --> Phase_E: 毕业达标
    Phase_E --> [*]: 通关

    note right of Phase_A
        毕业条件：
        - A方案完成率 ≥ 90%
        - B方案完成率 ≥ 80%
        - 阅读速度达标
    end note

    note right of DayComplete
        昨日未完成 →
        今日课程 = 昨日续学
    end note
```

## 五、动态配置中心

> **核心原则：阶段标准、依赖关系、训练内容不写在代码里，运营可实时调整。**

### 配置层次

```mermaid
graph LR
    subgraph 运行时加载
        YAML["training_curriculum.yaml<br/>首日课表 / 推进策略"]
        YAML2["training_advance_rules.yaml<br/>毕业阈值 / 达标条件"]
    end

    subgraph 数据库
        CT["ContentItem 表<br/>课程元数据（阶段/依赖/时长）"]
    end

    YAML --> RULES["规则引擎读取"]
    YAML2 --> RULES
    CT --> PLAN["方案生成器"]

    style YAML fill:#fff9c4,stroke:#f9a825
    style YAML2 fill:#fff9c4,stroke:#f9a825
    style CT fill:#e1bee7,stroke:#8e24aa
```

### 关键配置示例

**阶段毕业规则** (`training_advance_rules.yaml`)：
```yaml
advance_rules:
  phase_A_to_B:
    conditions:
      - type: "completion_rate"
        threshold: 0.9
        scope: "a_lessons"
      - type: "completion_rate"
        threshold: 0.8
        scope: "b_lessons"
      - type: "reading_speed"
        threshold: 800
        unit: "字/分钟"
    on_pass: "advance_to_phase_B"
    on_fail: "supplement_phase_A_basics"
```

**首日课表** (`training_curriculum.yaml`)：
```yaml
day_one:
  a_lessons:
    - skill: "超脑阅读"
      stage: 1
      part: 1
  b_lessons: []

after_day_one:
  strategy: "random"  # 次日开始随机选课
```

## 七、LLM / Agent / RAG 定位

```mermaid
graph TB
    subgraph CORE["🔒 核心调度（确定性逻辑）"]
        SM["状态机引擎"]
        RE["规则引擎"]
        BP["背包算法"]
    end

    subgraph LLM["🤖 LLM 增强（非核心）"]
        COLD["冷启动推荐<br/>RAG 检索课程标签"]
        EXPLAIN["反馈解释<br/>昨日打卡效果/备注回流"]
        COACH["训练引导语 / 答疑教练提示复用"]
    end

    CORE -->|"触发时机"| EXPLAIN
    CORE -->|"触发时机"| COACH
    CORE -->|"冷启动"| COLD

    style CORE fill:#e8f5e9,stroke:#2e7d32,stroke-width:3px
    style LLM fill:#fff3e0,stroke:#ef6c00
```

> **原则：核心调度不走 LLM。** 状态流转、进度推进、课程排序交给确定性规则引擎。LLM 用于文案、答疑辅导与昨日反馈解释；昨日 `result/note` 与答疑 `mistake_pattern` 已接入（见 [数据闭环与预留说明.md](数据闭环与预留说明.md)）。
